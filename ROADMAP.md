# Roadmap

This roadmap describes the project as an early-stage learning experiment. Planned items are not completed features.

## Current Baseline

- A CET-6 reading analysis workflow exists for a small sample.
- Reading Section C has reviewed final/display output for selected questions.
- Reading Section B has a completed sample chain from source extraction to display/export artifacts.
- CLI commands exist for config validation, source integrity checks, display validation, and Markdown export.
- The project already records conservative boundaries around scope, manual review, and source integrity.

## Near-Term Priorities

- Keep public examples fictional and safe to publish.
- Add a question-level analysis schema for future tools and PWA integration.
- Improve public-facing documentation around positioning, copyright, and workflow.
- Document which artifacts are sample-only and which are reusable patterns.
- Add lightweight checks for mock example JSON and schema validity.

## Medium-Term Exploration

- Register and validate Reading Section B through the config/CLI workflow.
- Design a Reading Section A sample workflow before broadening support.
- Aggregate Reading Section B and Section C output into a single reading-review view.
- Define a high-yield vocabulary scoring model based on role, not frequency alone.
- Design a local-first wrong-question review PWA data model.

## Longer-Term Ideas

- Support user-provided text import without bundling copyrighted source material.
- Add spaced-review scheduling for wrong questions and high-yield items.
- Explore listening transcript analysis when legally provided transcripts are available.
- Add export formats for Anki-like cards or other personal study tools.
- Add CI checks for schemas, examples, and documentation links.

## Non-Goals For Now

- No full exam-content database.
- No official dataset claim.
- No score-improvement promise.
- No automatic final-explanation factory.
- No broad multi-year support until each sample path is reviewed.
